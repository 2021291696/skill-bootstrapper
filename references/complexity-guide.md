# 复杂度判断详细指南

## 六个维度详解

### 1. 步骤/分支数
- 简单（1分）：线性流程，≤3 步。如 memory-keeper（记住→归位 / 忘掉→删除）
- 复杂（2分）：≥4 步或多条件分支。如 weaver（5 步 + 权限 + 经验 + 趋势 + 自迭代）

### 2. 依赖
- 简单（1分）：纯 Markdown，skill 本身不需要外部工具
- 复杂（2分）：依赖代码执行/脚本/MCP 服务/外部 API

### 3. 触发词
- 简单（1分）：1-2 个明确短语，不太会误触发
- 复杂（2分）：多种上下文需精确匹配，需仔细调优 description

### 4. 输出文件
- 简单（1分）：输出 1-2 个文件，格式固定
- 复杂（2分）：多文件、多格式、多阶段输出

### 5. 主观性
- 简单（1分）：输出可客观验证（文件存在、格式正确、链接有效）
- 复杂（2分）：输出质量靠人判断（写作风格、设计品味、语义准确性）

### 6. 参考经验
- 简单（1分）：类似 skill 已存在，主要是改编
- 复杂（2分）：全新领域，无现成参考

## 综合判定

| 总分 | 判定 | 通道 |
|------|------|------|
| 6-8 分 | 简单 | 快速通道（writing-plans → subagent） |
| 9-12 分 | 复杂 | Eval 通道（skill-creator） |

## 示例

### memory-keeper — 简单（7分）
| 维度 | 得分 | 理由 |
|------|------|------|
| 步骤 | 1 | 记住/忘掉各一个主线 |
| 依赖 | 1 | 纯 Markdown |
| 触发词 | 1 | 记住/忘掉两类，明确 |
| 输出 | 2 | 涉及 memory + settings.json + CLAUDE.md |
| 主观性 | 1 | 写入位置可客观验证 |
| 参考 | 1 | 借鉴了 global-tidy 的分层逻辑 |

### weaver-自我迭代 — 复杂（11分）
| 维度 | 得分 | 理由 |
|------|------|------|
| 步骤 | 2 | 5 步 + 权限 + 经验 + 趋势 + 自迭代 |
| 依赖 | 1 | 纯 Markdown |
| 触发词 | 2 | 10+ 触发词，多种上下文 |
| 输出 | 2 | 多文件（memory + CLAUDE.md + settings.json） |
| 主观性 | 2 | 经验发现、趋势追踪靠判断 |
| 参考 | 2 | 全新设计 |
